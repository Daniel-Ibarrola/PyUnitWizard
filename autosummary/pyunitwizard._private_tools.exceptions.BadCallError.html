pyunitwizard._private_tools.exceptions.BadCallError

exception pyunitwizard._private_tools.exceptions.BadCallError(argument=None)[source]

Exception raised when a method, or a class, was not properly called or instantiated.

This exception is raised when a method or a class was not properly called or instantiated.

Parameters

argument (str, optional) – The name of the possible wrong input argument.

Raises

BadCallError – A message is printed out with the name of the class or the method raising the exception, the possible wrong argument, the link to the API documentation, and the link to the issues board of PyUnitWizard’s GitHub repository.

Examples

>>> from pyunitwizard._private_tools.exceptions import BadCallError
>>> def method_name(item, a=True):
...    if type(a) not in [int, float]:
...       raise BadCallError('a')
...    pass
